Using Git

1. Make sure that git bash is setup in path. mkdir where u want the git repository to be initialized in.

2. "git init" in given folder

3. "git branch [name of branch]" to create and enter a branch other than master, this however will require the existence of files, you cannot enter a branch without the presence of any files. "git branch" will return you to the master branch. a "-v" will provide more information on given branch.

4. "git config --list" will present all the settings

5. "git config user.name 'Ali Rassolie' --global" for

6. "git config user.emali '[email address]'"

7. "git status" to see untracked files etc

8. "git add [untracked file]"

9. "git commit -m [message associated with given commit]"

10. "git checkout [branch]" to enter given branch

11. "git push" to push the commits to the cloud

A pull request is a request to the owner of another repo to take changes you made in a branch on your fork of their repo and integrate it into theirs as if you had done the work on theirs directly. It is a request for the owner of a repository to accept your changes, that you made on your own copy of the repo ("your fork"), and "pull" them into the owner's repository.
